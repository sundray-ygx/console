# 个人学习系统使用说明

## 功能概述

基于 PARA 知识库框架的个人学习积累系统，支持：

1. **书籍索引** - 自动扫描 NAS 电子书目录
2. **内置阅读器** - PDF + Markdown，注意力连续
3. **三层笔记** - 章节笔记 + 读后感 + 文本高亮
4. **学习进度** - 自动记录阅读位置
5. **在线搜索** - 按书名搜索电子书
6. **开源课程** - GitHub 开源课程配置（Phase 2）

---

## 快速开始

### 第一步：打开 Areas 页面

从侧边栏导航到 **Areas** 页面。

### 第二步：切换到「课程」Tab

页面上方有标签页切换，点击 **📚 课程**。

### 第三步：扫描书籍

点击 **🔄 扫描书籍** 按钮，系统会自动扫描 NAS ` /volume1/books/ ` 目录中的所有 PDF 电子书。

> ⏱ 扫描耗时取决于电子书数量，完成后页面会显示最新添加的书籍列表

### 第四步：浏览和搜索

| 功能 | 操作 |
|------|------|
| **浏览最近书籍** | 扫描完成后自动显示 |
| **搜索书籍** | 在搜索框输入书名关键词 |
| **打开阅读** | 点击任意书籍卡片 |

### 第五步：阅读和笔记

点击书籍卡片打开 **CourseReader 阅读器**：

```
┌─────────────────────────────────────┐
│  ✕ 书名                📖 42% ◀ ▶ │  ← 顶部工具栏
├──────────────────────┬──────────────┤
│                      │ 📝 笔记 (1)  │
│   PDF/Markdown       │ 🎨 高亮 (3)  │
│   阅读区域            │              │
│                      │ ──── 章节笔记│
```

**阅读器功能：**

| 区域 | 按键/操作 | 功能 |

## 笔记系统

### 章节笔记
- 阅读时记录该章节的要点、疑问、总结
- 支持 Markdown 语法（粗体、列表、代码等）
- **自动保存**：输入后 3 秒自动保存

### 读后感
- 读完一本书后的整体感悟
- 支持 Markdown 语法
- 5 秒自动保存

### 文本高亮 🎨
- 选中任意文本自动触发高亮
- 高亮内容会显示在「高亮」Tab 中
- 支持删除高亮（未来版本支持颜色选择）

## 后端 API

| 方法 | 路径 | 说明 |
|------|------|------|
| `POST` | `/api/books/index` | 触发扫描 NAS 书籍目录 |
| `GET` | `/api/books/search?q=` | 搜书 |
| `GET` | `/api/books/recent?limit=` | 最近索引 |
| `GET` | `/api/books/{id}/info` | 书籍信息 |
| `GET` | `/api/books/{id}/view` | 在线阅读（文件流） |
| `GET` | `/api/learning/progress/{course_id}` | 获取学习进度 |
| `PUT` | `/api/learning/progress` | 更新学习进度 |
| `GET` | `/api/learning/notes/{course_id}` | 获取笔记 |
| `POST` | `/api/learning/notes` | 创建笔记 |
| `PUT` | `/api/learning/notes/{id}` | 更新笔记 |
| `DELETE` | `/api/learning/notes/{id}` | 删除笔记 |
| `GET` | `/api/learning/highlights/{course_id}` | 获取高亮 |
| `POST` | `/api/learning/highlights` | 创建高亮 |
| `DELETE` | `/api/learning/highlights/{id}` | 删除高亮 |

## 常见问题

**Q: 点击"扫描书籍"后没有任何变化？**
A: 首次扫描可能需要几秒钟到几分钟（取决于书籍数量）。完成后页面会自动刷新显示书籍列表。

**Q: 搜索不到我想要的书籍？**
A: 搜索是模糊匹配书名。如果确定 NAS 中有该文件，尝试搜索书名的一部分。

**Q: PDF 阅读器加载很慢？**
A: 大文件 PDF 需要更多加载时间。进度条会显示加载状态。

**Q: 笔记保存在哪里？**
A: 所有笔记保存在后端 SQLite 数据库中（`console-api/data/console.db`），仅本地存储，不会同步到 Notion。

**Q: 支持哪些电子书格式？**
A: Phase 1 仅支持 PDF。EPUB 支持计划在 Phase 2 中实现。
